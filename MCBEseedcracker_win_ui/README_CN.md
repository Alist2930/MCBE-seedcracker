# MCBE Seed Cracker UI

[English](README.md) | 简体中文

Windows 桌面应用程序版本，提供图形化界面，无需命令行操作。

---

## 功能特性

- ✅ **图形化界面** - 直观的 Windows 桌面应用
- ✅ **低32位破解** - 使用建筑坐标破解种子低32位
- ✅ **高32位破解** - 使用群系样本破解种子高32位
- ✅ **进度保存/恢复** - 支持中断后继续破解
- ✅ **中英文切换** - 支持中文和英文界面
- ✅ **多版本支持** - 支持 MC 1.18/1.19/1.20/1.21
- ✅ **建筑图标** - 建筑类型显示对应图标
- ✅ **群系颜色** - 群系类型显示对应颜色

## 使用说明

### 1. 低32位破解（建筑坐标）

1. 在游戏中收集建筑坐标（推荐 5 个不同类型的建筑）
2. 点击「添加建筑」按钮，输入建筑类型和坐标
3. 选择破解范围（测试模式 0-100M 或全量模式 0-4.3B）
4. 点击「开始破解」
5. 等待破解完成，查看候选低32位值

**支持的建筑类型：**

| 建筑类型         | 中文名       | 分布类型   |
| ---------------- | ------------ | ---------- |
| desert_temple    | 沙漠神殿     | linear     |
| swamp_hut        | 女巫小屋     | linear     |
| jungle_temple    | 丛林神庙     | linear     |
| shipwreck        | 沉船         | linear     |
| ocean_ruins      | 海底废墟     | linear     |
| igloo            | 雪屋         | linear     |
| ocean_monument   | 海底神殿     | triangular |
| end_city         | 末地城       | triangular |
| village          | 村庄         | triangular |
| mansion          | 林地府邸     | triangular |
| ancient_city     | 深暗古城     | triangular |
| pillager_outpost | 掠夺者前哨站 | triangular |
| nether_fortress  | 下界要塞     | linear     |
| bastion_remnant  | 堡垒遗迹     | linear     |

> **提示**：优先使用 linear 类型建筑（如沙漠神殿、女巫小屋），计算量更小，破解更快。

**建筑定位区块确定方法：**

- **沙漠神殿**：中心位置所在的区块
- **海底神殿**：中心位置所在的区块
- **女巫屋**：建筑占区块面积最大的区块
- **丛林神庙**：建筑占区块面积最大的区块
- **末地城**：入口潜影贝方形结构占区块面积最大的区块
- **沉船**：完整沉船取船头所在区块（船头大概是刚好顶到区块边界的那端），残缺沉船取船占区块面积最大的区块

### 2. 高32位破解（群系样本）

1. 在游戏中收集群系样本坐标（推荐 5 个不同群系）
2. 选择 MC 版本（1.18/1.19/1.20/1.21）
3. 点击「添加群系」按钮，输入坐标和群系类型
4. 输入低32位值（从低32位破解结果获取）
5. 点击「开始破解」
6. 等待破解完成，查看完整种子

**采样建议：**

- 采样高度建议 Y >= 200（避免地下群系干扰）
- 选择稀有群系（如樱花林、苍白之园）效果更好
- 避免选择常见群系（如平原、海洋）
- **选择群系中心区域的坐标**，远离群系边界至少3格以上

### 3. 验证种子

破解完成后，在 [ChunkBase](https://www.chunkbase.com/apps/seed-map) 验证种子是否正确。

> **注意**：ChunkBase 等网站使用 Java 版群系名称，与基岩版不同。例如：Java 的 `stony_shore` 在基岩版是 `stone_beach`，Java 的 `dark_forest` 在基岩版是 `roofed_forest`。验证时请注意区分。

## 版本兼容性

### ⚠️ 重要限制

**高32位破解功能基于 cubiomes 库，仅支持到 Java 版 1.21.3，之后不再更新。**

| cubiomes 信息  | 详情                         |
| -------------- | ---------------------------- |
| 最新版本       | 4.1.2                        |
| 最后更新       | 2024年11月10日               |
| 支持的最高版本 | Java 版 1.21.3 (Pale Garden) |

**cubiomes 更新状态：**

- cubiomes 在 2024年11月发布 4.1.2 版本后**停止更新**
- 不支持 Java 版 1.22+ 或基岩版 1.22+
- 不支持 2025年及之后的 Minecraft 版本

### 版本对应关系

| MC 版本 | 新增群系                 |
| ------- | ------------------------ |
| 1.18    | 溶洞、繁茂洞穴、山地群系 |
| 1.19    | 深暗古城、红树林沼泽     |
| 1.20    | 樱花林                   |
| 1.21    | 苍白之园                 |

### 基岩版 vs Java 版差异

即使版本号相同，Java 版和基岩版的群系生成也存在差异：

- **Y 轴群系变化**：Java 版群系在 Y 轴上变化显著，基岩版较稳定
- **群系边界**：两版本的群系边界位置可能略有差异
- **新版本差异**：基岩版 1.26.x 与 Java 版 1.21 群系算法有较大差异

**建议**：使用基岩版 1.21.0 版本，与 cubiomes 支持的 Java 版 1.21 群系算法最接近。

## 常见问题

### 低32位破解失败

**可能原因：**

1. 建筑坐标错误
2. 建筑数量不足（建议至少 5 个）
3. 建筑类型选择不当（优先使用 linear 类型）

**解决方案：**

- 核实坐标是否正确
- 增加建筑数量
- 更换建筑类型

### 高32位破解失败

**可能原因：**

1. 低32位值错误
2. 群系样本坐标或 ID 错误
3. 采样高度不当（建议 Y >= 200）
4. 版本不兼容（基岩版 1.26.x 与 Java 版 1.21 差异）

**解决方案：**

- 确认低32位值正确
- 核实群系样本
- 提高采样高度
- 使用基岩版 1.21.0 版本
